# 战术方案编写指南

## 工作区布局

- 战术方案页右下角支持自由缩放；放大窗口只会增加可视区域，不会改字号、行高、列宽和单元格大小。
- 左侧解析区和右侧文本区之间新增可拖拽分隔线；拖动时会实时调整两侧宽度比例，双击可回到 50%。
- 窗口大小、位置、分隔线比例和上次关闭时所在标签页都会自动记忆；设置页中的「重置战术方案布局」可恢复默认布局。
- 设置页始终保持默认大小；只有战术方案页支持缩放。

## 三条核心规则

### 规则一：`{time:分:秒}` 决定什么时候播报

```
{time:00:10} ...    ← 开战后第 10 秒
{time:01:30} ...    ← 开战后第 1 分 30 秒
{time:90}   ...     ← 开战后第 90 秒（和上面一样）
{time:00:30,p2} ... ← P2 开始后第 30 秒
```

每一行必须以 `{time:XX}` 开头，没有 `{time:XX}` 的行会被忽略。

如果你想按阶段相对时间播报，可以在时间后面加逗号和阶段标签：

```
{time:00:10,p1} {所有人}开场分散
{time:00:08,i1} {所有人}过渡准备
{time:00:20,p2} {所有人}P2 集合
{time:00:12,p1r2} {所有人}第二轮 P1 集合
```

### 规则二：`{谁}` 决定谁能听到

```
{time:00:10} {所有人}全员分散
{time:00:20} {坦克}拉好仇恨
{time:00:30} {张三}你去打断
```

`{谁}` 后面紧跟的文字，就是那个人会听到的内容。

### 规则三：`{谁}` 前面的文字是备注，不会播报

```
{time:00:32} 亡者吐息1:{所有人}去左边头前
```

这里 `亡者吐息1:` 写在 `{所有人}` 前面，它只是给你自己看的备注标签，不会被念出来。所有人只会听到「去左边头前」。

## `{谁}` 可以写什么？

### 全团

| 写法 | 效果 |
|------|------|
| `{所有人}` | 全团都听到 |
| `{全团}` | 同上 |
| `{all}` | 同上（英文） |
| `{everyone}` | 同上（英文） |

示例：`{time:00:10} {所有人}全员分散`

### Boss 技能（只看不播报）

| 写法 | 效果 |
|------|------|
| `{BOSS}` | 显示在时间轴面板，但不会播报给任何人 |
| `{boss}` | 同上 |

`{BOSS}` 比较特殊——写了它的行会在时间轴面板上显示为一个单元格，但不会匹配任何玩家，没人会听到语音播报。

**用途：** 在面板上标出 Boss 技能释放时间，方便对照。

```
{time:00:07} {BOSS}{spell:1258610}     ← 面板上显示"青龙下凡"的技能图标和名称
{time:00:14} {BOSS}{spell:1262289}     ← 面板上显示"孤注一掷"
{time:00:14} {小德1}豹奔               ← 小德1 听到"豹奔"
```

前两行 `{BOSS}` 只在面板上显示（带技能图标），不播报；第三行 `{小德1}` 正常播。内置模板已自带 `{BOSS}`，你自己写方案时也可以手动加。

### 指定玩家

直接写角色名：

```
{time:00:20} {张三}你去打断
```

只有叫「张三」的玩家会听到「你去打断」，其他人什么都听不到。

### 坦克 / 治疗 / 输出

| 写法 | 谁听到 |
|------|--------|
| `{tank}` 或 `{坦克}` | 所有坦克 |
| `{healer}` 或 `{heal}` 或 `{治疗}` | 所有治疗 |
| `{dps}` 或 `{dd}` 或 `{输出}` | 所有输出 |

示例：`{time:00:15} {healer}准备刷血`

### 指定职业

| 写法 | 写法（中文） |
|------|-------------|
| `{warrior}` | `{战士}` |
| `{paladin}` | `{圣骑士}` |
| `{hunter}` | `{猎人}` |
| `{rogue}` | `{盗贼}` |
| `{priest}` | `{牧师}` |
| `{deathknight}` | `{死亡骑士}` |
| `{shaman}` | `{萨满祭司}` |
| `{mage}` | `{法师}` |
| `{warlock}` | `{术士}` |
| `{monk}` | `{武僧}` |
| `{druid}` | `{德鲁伊}` |
| `{demonhunter}` | `{恶魔猎手}` |
| `{evoker}` | `{唤魔师}` |

示例：`{time:00:30} {法师}释放时光变速`

### 指定专精

比职业更精确。可以直接写专精名称：

| 写法 | 谁听到 |
|------|--------|
| `{戒律}` 或 `{discipline}` | 戒律牧师 |
| `{神圣}` 或 `{holy}` | 神圣牧师 **和** 神圣圣骑（两个都叫「神圣」） |
| `{恢复}` 或 `{restoration}` | 恢复德鲁伊 **和** 恢复萨满 |
| `{防护}` 或 `{protection}` | 防护战士 **和** 防护圣骑 |
| `{冰霜}` 或 `{frost}` | 冰霜法师 **和** 冰霜死骑 |

所有 39 个专精都支持，中英文均可。上面只列了容易重名的，其余如武器、狂怒、惩戒、奇袭、暗影、鲜血、元素、奥术、火焰、酒仙、踏风、平衡、浩劫、湮灭、恩护、增辉等同样可用。

**重名专精怎么区分？** 用加号组合职业名消歧：

```
{time:00:10} {神圣+牧师}开守护之魂    ← 仅神圣牧师，圣骑不受影响
{time:00:20} {神圣+圣骑士}圣佑术      ← 仅神圣圣骑
{time:00:30} {防护+战士}盾墙          ← 仅防护战士，不是防护骑
{time:00:40} {冰霜+法师}冰脉          ← 仅冰霜法师，不是冰霜死骑
{time:00:50} {治疗+牧师+戒律}开减伤    ← 三重条件：治疗职责 且 牧师职业 且 戒律专精
```

> **小贴士：** 只写 `{神圣}` 不加职业名，神圣牧师和神圣圣骑都会听到——这不是 Bug，是设计。如果你想两个都喊，直接写 `{神圣}` 最省事。

### 近战 / 远程

| 写法 | 谁听到 |
|------|--------|
| `{melee}` 或 `{近战}` | 所有近战专精 |
| `{ranged}` 或 `{远程}` | 所有远程专精 |

示例：`{time:00:45} {近战}远离Boss`

### 指定小队

| 写法 | 谁听到 |
|------|--------|
| `{g1}` | 第 1 小队 |
| `{g2}` | 第 2 小队 |
| … | … |
| `{g8}` | 第 8 小队 |

示例：`{time:01:00} {g1}去左边 {g2}去右边`

### 组合条件

**逗号 = 或（OR）**，满足任意一个就能听到：

```
{time:00:20} {tank,g1}开怪    ← 坦克 或 第1小队的人都能听到
```

**加号 = 且（AND）**，必须同时满足才能听到：

```
{time:00:10} {治疗+德鲁伊}开宁静    ← 必须同时是治疗专精 且 德鲁伊职业
{time:00:20} {治疗+武僧}开复苏之雾  ← 必须同时是治疗专精 且 武僧职业
```

**混合写法**，加号优先级高于逗号：

```
{time:00:30} {坦克+g1,治疗}注意    ← (坦克 且 第1小队) 或 全部治疗
```

| 写法 | 语义 |
|------|------|
| `{治疗,德鲁伊}` | 治疗 **或** 德鲁伊 |
| `{治疗+德鲁伊}` | 治疗 **且** 德鲁伊 |
| `{神圣+牧师}` | 神圣专精 且 牧师职业（= 神圣牧师） |
| `{治疗+牧师+戒律}` | 治疗职责 且 牧师 且 戒律专精 |
| `{坦克+g1,治疗}` | (坦克 且 1队) 或 治疗 |
| `{远程+近战}` | 不可能同时满足，无人听到（不报错） |

## 法术名自动转换

写 `{spell:法术ID}` 会自动变成法术的名字，方便你不用手打技能名：

```
{time:00:03} {奶德1} {spell:2825}     ← 自动变成"英勇"
{time:00:13} {奶德1} {spell:33891}    ← 自动变成"生命之树"
{time:00:27} {奶德1} {spell:740}      ← 自动变成"宁静"
```

**法术 ID 怎么查？** 去 [https://www.wowhead.com/](https://www.wowhead.com/) 搜技能，网址里的数字就是 ID。比如 `wowhead.com/spell=740` 里的 `740` 就是宁静的法术 ID。

## 一行里给不同人说不同的话

同一行可以写多个 `{谁}`，每个人只听到自己那段：

```
{time:00:32} 亡者吐息1:{所有人}被点名恐惧去左边头前 {萨满1}风行
```

| 玩家 | 听到什么 |
|------|----------|
| 所有人 | 「被点名恐惧去左边头前」 |
| 萨满1 | 「风行」 |
| — | 「亡者吐息1:」没人听到（这是备注标签） |

## 人员映射（推荐用法）

在方案顶部加一个 `[人员]` 区块，给角色起别名。这样换人只需改一处：

```
[人员]
坦克1=张三
奶德1=小明
萨满1=老王

{time:00:03} - {奶德1} {spell:2825}
{time:00:13} - {奶德1} {spell:33891}
{time:00:27} - {奶德1} {spell:740}
{time:00:32} 亡者吐息1:{所有人}去左边 {萨满1}风行
```

这里 `{奶德1}` 会自动替换成 `{小明}`，只有小明能听到对应内容。

## 高级/兼容用法：给未内置 Boss 补阶段规则

如果某个 Boss 还没有内置阶段识别，但你又想写 `{time:00:30,p2}` 这类阶段相对时间，可以在 `[时间轴]` 前面加一个 `[设置]` 区块。

> 宇宙之冕、奇美鲁斯已有内置阶段识别，不需要写 `[设置]`。

```
[设置]
i1={on:spell:1234569}
p2={on:spell:1238206}
i2={on:spell:1245874}
p3={on:spell:1238843}
```

当 Boss 释放指定技能时，插件会将其识别为阶段切换。已有内置阶段识别的 Boss，内置规则优先，这里的配置作为备用。

写法固定为：`阶段标签={on:spell:技能ID}`

- 阶段标签可以写 `p1`、`i1`、`p2`，也可以写 `p1r2`、`p2r2` 这种「第 N 轮阶段」格式；请使用小写。
- `[设置]` 里的 spell 语法仍然兼容，但 12.0 下内置阶段优先生效，不建议作为唯一手段。
- `[设置]` 只影响阶段相对时间，不影响普通 `{time:分:秒}`。
- 如果聊天提示「阶段 X 未检测到，N 条事件等待中」，说明自动阶段识别没接上，需要补正确的阶段切换规则。

## 一个位置多个候选人

如果某个位置经常换人（比如同一个人有好几个号），用逗号隔开：

```
[人员]
奶德1=小明主号,小明小号,小明三号
```

插件会自动找第一个当前在团队里的人。小明主号在就用主号，主号不在就找小号，以此类推。

## 从 lorrgs.io 粘贴

lorrgs.io 导出的格式可以直接用，只需在顶部加 `[人员]`：

```
[人员]
奶德1=你的角色名

{time:00:03} - {奶德1} {spell:2825}
{time:00:13} - {奶德1} {spell:33891}
{time:00:27} - {奶德1} {spell:740}
{time:00:53} - {奶德1} {spell:29166}
{time:01:29} - {奶德1} {spell:47788}
```

中间的 `-` 不需要删，它在 `{奶德1}` 前面，自动算备注不播报。

## 个人方案：直接粘贴，不需要名字

如果你只想给自己用，可以把 lorrgs.io 的内容直接粘到个人方案里，连名字都不用写：

```
{time:00:16} -  {spell:359816}
{time:01:15} -  {spell:363534}
{time:02:25} -  {spell:359816}
{time:04:18} -  {spell:359816}
```

个人方案里没有 `{谁}` 的行会自动对你触发，`-` 分隔符会被自动清理，播报时只念技能名。

> **注意：** 这条只限个人方案。团队方案里没有 `{谁}` 的行照样不播报。

如果你想在个人方案里给不同角色写不同的内容（比如一套方案给治疗号用、一套给坦克号用），仍然可以加条件：

```
{time:08:38} -  {恩护}{spell:363534}
{time:08:38} -  {治疗+德鲁伊}{spell:363534}
```

带了 `{谁}` 的行照常按条件过滤，跟以前一样。

## 不播报的行（注释）

没有 `{谁}` 的行不会播报任何内容，也不会显示在左侧时间轴面板上。可以当注释用：

```
{time:00:00} === P1 开始 ===
{time:00:10} {所有人}全员集合
{time:01:00} === P2 转阶段 ===
{time:01:05} {tank}拉Boss到中间
```

`=== P1 开始 ===` 和 `=== P2 转阶段 ===` 没有 `{谁}`，不会播报，也不会在时间轴面板上显示。它们只存在于右侧的文本编辑器中，方便你看结构。

面板上只出现有 `{谁}` 的行；没有 `{谁}` 的行是纯注释，只在编辑器里能看到。没有 `{time:XX}` 的行同样被忽略。

## 颜色文本

你可以用 WoW 标准颜色代码给文字上色：

```
{time:01:52} {spell:1246749} {小野}|cff3fc7eb祝福|r
```

`|cff3fc7eb` 是颜色开头（`3fc7eb` 是十六进制颜色值），`|r` 是颜色结束。中间的「祝福」会在时间轴面板和屏幕提示上显示为对应颜色。

TTS 语音播报时颜色代码会被自动剥离，只念纯文本「祝福」。

**常用颜色示例：**

| 写法 | 效果 |
|------|------|
| `\|cffff0000红色文字\|r` | 红色（#FF0000） |
| `\|cff00ff00绿色文字\|r` | 绿色（#00FF00） |
| `\|cff3fc7eb青色文字\|r` | 青色（#3FC7EB） |
| `\|cffffcc00金色文字\|r` | 金色（#FFCC00） |

> **格式固定：** `|cff` + 6 位十六进制颜色 + 文字 + `|r`。`cff` 中的 `ff` 是透明度（固定写 `ff` 即可），后面 6 位才是颜色。

## 静默标记（显示但不念）

如果某段文字你想显示在屏幕上但不想被语音念出来，用尖括号或波浪线包起来：

```
{time:00:10} {所有人}分散 <站位参考P1图>
{time:00:20} {tank}换坦 ~~注意仇恨~~
```

- `<站位参考P1图>` → 屏幕上看得到，但语音不会念
- `~~注意仇恨~~` → 同上

## 自定义配音（语音包系统）

STT 支持语音包：播报文本精确匹配到语音包内的 mp3 文件名时，播放真人配音替代系统 TTS。

原理很简单：`{spell:ID}` 在解析阶段会被替换成技能名，到播报时文本已经是「宁静」而不是 `{spell:740}` 了。STT 拿这个文本去当前语音包目录下找同名 mp3，找到就播，找不到就走 TTS。

```
{time:01:30} {healer}{spell:740}
```

上面这行：`{spell:740}` 解析后变成「宁静」→ 匹配到语音包内 `宁静.mp3` → 播放真人配音。直接写 `宁静` 两个字效果完全一样。

### 使用语音包

1. 安装语音包插件（和装普通插件一样，丢到 AddOns 目录）
2. `/reload`
3. 设置 → 语音 → 自定义配音 → 下拉选择语音包

STT 出厂自带默认语音包（宁静、升腾、回溯等治疗大招），开箱即用。

### 制作自己的语音包

语音包就是一个独立插件，只需要 TOC 文件 + mp3 音频，不需要写任何代码：

```
STT_VoicePack_SeSe/
├── STT_VoicePack_SeSe.toc
├── 宁静.mp3
├── 升腾.mp3
└── ...
```

**TOC 文件内容（完整示例）：**

```
## Interface: 120000
## Title: STT语音包 - 瑟瑟
## Notes: STT 自定义配音语音包
## Dependencies: ShengTangTools
## X-STT-VoicePack-Name: 瑟瑟语音包
## X-STT-VoicePack-Path: Interface\AddOns\STT_VoicePack_SeSe\
```

mp3 文件名必须和游戏内技能名一模一样。放到 AddOns 目录后 `/reload`，设置里就能看到你的语音包了。

没匹配到配音的文本正常走 TTS，不受影响。

坦克1=大壮
坦克2=铁柱
奶骑1=豆豆
奶德1=小明,小明小号

{time:00:10} P1旋风斩:{近战}远离Boss
{time:00:13} - {奶德1} {spell:33891}
{time:00:20} {所有人}全员分散
{time:00:27} - {奶德1} {spell:740}

## 常见问题

**Q：我写了一行但没有播报？**
检查两点：1) 是否有 `{time:XX}`；2) 是否有 `{谁}` 指定了受众。两个都要有才会播报。

**Q：怎么让所有人都听到？**
用 `{所有人}` 或 `{all}`。团队方案中不能省略——不写 `{谁}` 等于注释。个人方案中不写 `{谁}` 会自动对你触发。

**Q：`{time:00:30,p2}` 这种写法支持吗？**
支持。表示「P2 开始后第 30 秒播报」。宇宙之冕和奇美鲁斯已有内置阶段识别，直接写即可，不需要 `[设置]`。未内置的 Boss 需要在 `[设置]` 里补阶段规则。

**Q：`p1r2` 是什么意思？**
「第 2 轮进入 P1」。`{time:00:12,p1r2}` = 第二次进 P1 后 12 秒播报。不写 `rN` 默认第 1 轮，`p1` 等价于 `p1r1`。阶段标签用小写，`P1R2` 无效。

**Q：`[设置]` 是必须的吗？**
不是。已内置阶段识别的 Boss（宇宙之冕、奇美鲁斯）不需要写。只有未内置的 Boss 想用阶段相对时间才需要补。普通 `{time:分:秒}` 不受影响。

**Q：可以用 MRT 的 `{H}治疗内容{/H}` 吗？**
不支持这种块级写法，会被自动清理。请用 `{healer}治疗内容` 代替。

**Q：旧方案用的 `{{双括号}}` 还能用吗？**
能用，会自动转换成单括号。但建议新方案直接用单括号 `{}`。

**Q：中文标点可以用吗？**
可以。`{time：01：30}` 里的中文冒号 `：` 会自动转成英文 `:`。中文花括号 `｛｝` 和全角加号 `＋` 也会自动转换。

**Q：`{治疗+德鲁伊}` 和 `{治疗}{德鲁伊}` 有什么区别？**
`{治疗+德鲁伊}` 是 AND，必须同时满足两个条件。`{治疗}{德鲁伊}` 是两个独立花括号，等于 OR（治疗或德鲁伊都能听到）。

**Q：`{神圣}` 会匹配两个职业？**
对。神圣牧师和神圣圣骑都叫「神圣」，所以 `{神圣}` 两个都会听到。加职业名区分：`{神圣+牧师}` 只匹配神圣牧师，`{神圣+圣骑士}` 只匹配神圣圣骑。防护、冰霜、恢复同理。

**Q：`{BOSS}` 和 `{所有人}` 有什么区别？**
`{所有人}` 全团都会听到语音播报。`{BOSS}` 不会播报给任何人，只在时间轴面板上显示 Boss 技能时间点。

**Q：我的方案没有 `{BOSS}`，时间轴面板上看不到 Boss 技能了？**
对。面板只显示有 `{谁}` 的行，想看 Boss 技能时间线就加 `{BOSS}`。内置模板已自带。

**Q：没有 `{谁}` 的注释行为什么不显示了？**
面板只显示带 `{谁}` 的行。注释行仍在编辑器中，面板不显示。

## 触发轴（技能触发播报）

大秘境没有固定时间轴，适合用「技能触发」来决定什么时候播报。团本里如果某个 Boss 的节奏更适合按技能事件写，也可以直接用触发轴。

### 核心区别

| 维度 | 团本时间轴 | 触发轴 |
|------|-----------|--------|
| 行头 | `{time:分:秒}` | `{on:spell:技能ID}` |
| 触发方式 | 开战后第 N 秒 | Boss 释放指定技能时 |
| 适用场景 | 团本固定时间点 | 大秘境、团本事件点 |
| 其余规则 | `{谁}` + 内容 | 完全一样 |

只有行头不同，其余写法和团本一模一样。

### 格式

```
{on:spell:技能ID} {谁}播报内容
{on:spell:技能ID:-N} {谁}提前N秒播报内容
{on:spell:技能ID}#N {谁}第N次的播报内容
{on:spell:技能ID:-N}#N {谁}第N次提前N秒播报内容
```

- **`{on:spell:技能ID}`** — 触发标识，表示「当这个技能触发时」。不会被播报，只是告诉插件什么时候播。
- **`:-N`** — 可选，表示在技能真正触发前 N 秒就先播报一次。
- **`#N`** — 可选，指定第 N 次触发才播报。
- **`{谁}`** — 和团本一样，没有 `{谁}` 就不播报。

### 对照速查

| 写法 | 播报效果 |
|------|----------|
| `{on:spell:466064} {所有人}躲正面` | 全员听到「躲正面」 |
| `{on:spell:466064} {tank}嘲讽` | 坦克听到「嘲讽」 |
| `{on:spell:466064} 备注:{所有人}快躲` | 全员听到「快躲」（「备注:」不播报） |
| `{on:spell:466064}#2 {所有人}第二次集合` | 第2次触发播「第二次集合」 |
| `{on:spell:466064:-15} {治疗}预铺HOT` | 技能前15秒对治疗播「预铺HOT」 |
| `{on:spell:466064:-5}#2 {所有人}第二次还有5秒` | 第2次触发前5秒预警 |
| `{on:spell:466064} {所有人}{spell:466064}快躲` | 全员听到「技能名 快躲」 |
| `{on:spell:466064}` | 不播报（纯触发标记） |
| `{on:spell:466064} 这是备注` | 不播报（无 `{谁}`） |

### 提前预警

`:-N` 适合需要提前准备的场景，比如治疗预铺、减伤预读、换位预备。

- `{on:spell:466064:-15} {治疗}预铺`：技能前 15 秒提醒治疗准备。
- `{on:spell:466064} {所有人}注意大伤害`：技能真正触发时再补一句正式播报。
- 同一个技能可以同时写 advance 和 normal 规则，互不覆盖。
- 剩余时间不足 N 秒时，插件会立即播这条 advance 规则，不会丢掉。
- 没写 `:-N` 的行会继承设置里的「提前 X 秒播报」；写了 `:-N` 以行内值为准，不叠加全局秒数。

### `{on:spell:xxx}` 和 `{spell:xxx}` 的区别

这两个长得很像，但作用完全不同：

- **`{on:spell:466064}`** — 触发标识，写在行头，不播报，等于「当技能 466064 触发时」。
- **`{spell:466064}`** — 技能名替换，写在内容里，播报时自动变成技能的名字。

### 完整示例

```
[方案]
名称 = 迈萨拉洞窟 - 凡尔克斯
作者 = 玩家

[人员]
坦克1 = 张三

[触发轴]
{on:spell:466064:-15} {治疗}预铺HOT
{on:spell:466064} {所有人}躲正面
{on:spell:466556} {所有人}进罩子
{on:spell:465904} {坦克1}准备大减伤
{on:spell:466556}#2 {所有人}第二次进罩子
{on:spell:466556}#3 {所有人}最后一次进罩子
```

### 团本中使用触发轴

触发轴不限大秘境，团本也能用。某些技能按事件点播报更合适的，直接整份方案写成 `[触发轴]` 就行。固定时间点仍推荐 `[时间轴]`；两种格式二选一，不混写。`#N` 和 `:-N` 同样支持。

### 旧格式兼容

旧的管道符格式 `{spell:xxx}|模式|内容` 仍然可用，插件会自动识别。但新方案建议统一用 `{on:spell:xxx}` 格式——写法更简单，和团本规则完全一致。

## 鲁拉符文助手

鲁拉（L'ura）Boss 需要按顺序踩 5 个符文。打开 **设置 → 鲁拉符文助手**，点一次「创建符文宏」，再把 5 个符文按钮拖到动作条上就能直接用。

点击「创建符文宏」后，STT 会一次性创建或重建 5 个通用宏（◇STT、△STT、TSTT、○STT、XSTT）。这些宏写的就是当前设置下的真实命令，比如：

- **三角：** `/rw Interface/AddOns/ShengTangTools/media/rune_tran`
- **切到团队频道后：** `/raid Interface/AddOns/ShengTangTools/media/rune_tran`
- **切到自定义频道 4 后：** `/4 Interface/AddOns/ShengTangTools/media/rune_tran`

只要 ◇STT、△STT、TSTT、○STT、XSTT 已经存在，切换频道时，STT 就会自动把这 5 个宏改写成最新内容，所以：

- 切换频道后自动改写已有宏
- 设置页按钮继续显示 STT 自带的蓝色符文素材
- 拖拽时宏会临时切到稳定图标，落到动作条后自动恢复 STT 真图标
- 不需要重新创建宏
- 不需要重新拖动作条

### 5 个符文

| 编号 | 名称 | 拖拽后动作条效果 |
|------|------|-----------------|
| 1 | 菱形 | 发送菱形符文 |
| 2 | 三角 | 发送三角符文 |
| 3 | T形 | 发送 T 形符文 |
| 4 | 圆圈 | 发送圆圈符文 |
| 5 | 叉叉 | 发送叉叉符文 |

### 拖拽到动作条

1. 输入 `/st` 打开设置页
2. 进入 **鲁拉符文助手**
3. 点击「创建符文宏」
4. 在「符文快捷按钮」区域，把 5 个符文按钮左键拖到动作条
5. 开打时按踩符文的顺序依次点击

第一次用时先点「创建符文宏」就够了；后面继续拖同一个符文，会复用同名宏，不会重复占宏槽。拖拽时宏会短暂切到稳定图标，鼠标放下后再自动恢复成 STT 真图标。

> `/st rune macro` 旧命令仍保留，但现在只会提示你改用设置页拖拽。

### 不同频道的发送效果

在 **设置 → 鲁拉符文助手 → 符文频道** 里选频道。指挥和团员必须选同一个频道。已拖到动作条的 ◇STT、△STT、TSTT、○STT、XSTT 会自动跟着当前设置改写，不需要手动改宏内容。

默认是 **团队警告**，也就是 `/rw`。H 难度一般继续用这个就行，字够醒目，团长或助理来发最省事。M 难度如果打算让更多人自己发符文，建议全团统一切到 **团队**，也就是 `/raid`，这样没有团队助理权限的人也能发；如果你们还是想保留 `/rw` 的黄字提醒，那就给负责发符文的人助理权限。

| 频道 | 实际发送效果 | 说明 |
|------|-------------|------|
| 团队警告（默认） | `/rw <符文内容>` | 仅团长/助理可发；H 难度一般直接用这个 |
| 说 | `/say <符文内容>` | 附近玩家可见 |
| 团队 | `/raid <符文内容>` | 全团可见，任何团员可发；M 难度多人要发时更省心 |
| 自定义频道 | `/频道编号 <符文内容>` | 需先 `/join 频道名` 加入 |

> **注意：** encounter 中，STT 会自动额外监听 `/raid` 频道的消息，确保宏通道畅通。

### 自定义频道

如果团队警告权限不够（比如非团长/助理），可以用自定义频道：

1. 指挥和所有团员输入 `/join rune`（频道名随意）
2. 聊天窗口会显示类似 `[1. rune]`，这里的 `1` 就是频道编号
3. 设置 → 鲁拉符文助手 → 符文频道选「自定义频道」，编号拉到 `1`
4. 已拖到动作条的 5 个符文按钮会自动改为走 `/1` 发送，不需要重拖

> **注意：** 频道编号因人而异，取决于加入频道的顺序。`/chatlist` 可以查看当前所有频道和编号。

### 设置选项

| 选项 | 说明 |
|------|------|
| 接收符文显示 | 团员是否显示符文面板（默认开启） |
| 符文频道 | 发送和接收符文的聊天频道，指挥和团员必须一致 |
| 自定义频道编号 | 仅「自定义频道」时生效，填你的频道编号 |
